离线检测

navigator.onLine属性，值为true表示设备能上网，值为false表示设备离线。

两个事件：online和offline，当网络从离线变为在线或从在线变为离线时，分别触发这两个事件。

数据存储

Cookie

在客户端用于存储会话信息。
限制：
cookie在性质上绑定在特定的域名下的。
cookie存在客户端计算机上，还加入了一些限制确保cookie不会被恶意使用，同时不会占据太多磁盘空间。每个域的cookie总数是有限的。（IE6以及低版本20个，EI7及之后50个，FF50个，Opera30个，Safari和Chrome没规定）
浏览器对cookie的尺寸也有限制。最好将整个cookie的长度限制在4095B以内。尺寸限制影响的是一个域下的所有cookie，而并非每个cookie单独限制。
Cookie的构成

  1. 名称：一个唯一确定cookie的名称。cookie名称不区分大小写，名称必须经过URL编码
  2. 值：储存在cookie中的字符串值。值必须被URL编码
  3. 域：cookie对于哪个域是有效的。所有向该域发送的请求中都会包含这个cookie信息。如果没有明确设定，这个域会被认作来自设置cookie的那个域，对于指定域的所有子域都有效
  4. 路径：对于指定域中的那个路径，应该向服务器发送cookie。比如指定cookie只有从路径 http://www.ffj.com/books 访问，那么http://www.ffj.com 的页面就不会发送cookie信息，即使是同一个域，路径不同也不能访问
  5. 失效时间：表示cookie何时应该被删除的时间戳，即何时应该停止向服务器发送这个cookie。默认情况，浏览器会话结束时即将所有cookie删除；也可以自己设置删除时间，这样即使关闭浏览器cookie依然保存在用户的机器上，如果设置的是过去的时间，cookie会立即被删除。
  6. 安全标志：指定后，cookie只有在使用SSL连接时才发送到服务器，例如cookie信息只能发送给https://www.wrox.com 而http://www.wrox.com 的请求则不能发送cookie

每一段信息都作为Set-Cookie头的一部分，使用分号加空格分隔每一段，例如：
HTTP/1.1 200 OK
Content-Type: text/html
Set-Cookie: name=value; domain=.wrox.com; path=/; expires=Mon,22-Jan-17 07:10:24 GMT; secure
Other-header: other-header-value
注意：secure是cookie中唯一一个非名值对儿的部分，只包含一个secure单词
域，路径，失效时间和secure标志都是服务器给浏览器的指示，以指定何时应该发送cookie。这些参数并不会作为发送到服务器的cookie信息的一部分，只有名值对儿才会被发送。
JavaScript中的cookie
接口：BOM的document.cookie属性
document.cookie用来获取属性时，返回当前页面可用的所有cookie的字符串，一系列由分号隔开的名值对儿。所有名字和值都是经过URL编码的，必须用decodeURIComponent()来解码。
document.cookie用来设置新的cookie字符串时，这个cookie字符串会被解释并添加到现有的cookie集合中。设置document.cookie并不会覆盖cookie，除非设置的cookie的名称已经存在。
document.cookie=encodeURIComponent(“name”)+”=”+encodeURIComponent(“ffj”)+”; domain=.wrox.com; path=/“;

cookie操作：读取、写入和删除
子cookie

为了绕开浏览器的单域名下的cookie数限制，引入子cookie，它是存放在单个cookie中的更小段的数据，也就是使用cookie值来存储多个名称值对儿。
name=name1=value1&name2=value2&name3=value3
子cookie以查询字符串的格式进行格式化，这些值可以使用单个cookie进行存储和访问，而非对每个名称-值对儿使用不同的cookie存储。

如果担心开发中可能会达到单域名的cookie上限，子cookie是很好的备选方案，但要防止超过单个cookie的长度限制。
不要在cookie中存储重要和敏感数据，cookie数据并非存储在一个安全环境中，其中包含的任何数据都可以被他人访问。

IE用户数据

对IE用户数据的访问限制和对cookie的限制类似，要访问某个数据空间，脚本运行的页面必须来自同一个域名，在同一个路径下，并使用与进行存储的脚本同样的协议。
和cookie不同的是，你无法将用户数据访问限制扩展到更多的客户。用户数据默认是可以跨越会话持久存在的，同时也不会过期，数据需要removeAttribute()方法专门进行删除以释放空间。
和cookie一样，IE用户数据并非安全的，不能存储敏感信息。

Web存储机制

Web Storage两个目标：

  1. 提供一种在cookie之外存储会话数据的路径；
  2. 提供一种存储大量可以垮会话存在的数据的机制
    Storage类型
    clear():删除所有值
    key(index):获取index位置处的值的名字
    getItem(name):根据指定名字获取对应的值
    setItem(name,value):为指定的name设置对应的值value
    removeItem(name):删除由name指定的名值对儿
    Storage类型只能存储字符串。非字符串的数据在存储之前会被转换为字符串。
    sessionStorage对象
    sessionStorage对象存储特定于某个会话的数据，也就是该数据只保持到浏览器关闭。存储在sessionStorage中的数据可以跨越页面刷新而存在。
    sessionStorage对象是Storage的一个实例，故可以使用setItem()或直接设置新的属性来存储数据
    sessionStorage.setItem(“name”,”ffj”);
    sessionStorage.name=”ffj”;
    sessionStorage对象应该主要用于针对会话的小段数据的存储，如果需要跨越会话存储数据，可以选择globalStorage或localStorage。
    globalStorage
    globalStorage对象的目的是跨越会话存储数据，但有特定的访问限制。需要指定哪些域可以访问该数据。当使用globalStorage的时候，一定要指定一个域名。对于指定的域名，所有子域也可以访问。
    //保存数据
    globalStorage[“wrox.com”].name=”ffj”;
    //获取数据
    var name=globalStorage[“wrox.com”].name
    globalStorage对象不是Storage的实例，而具体的globalStorage[“wrox.com”]才是。这个存储空间对于wrox.com及其所有子域都是可以访问的。
    如果事先不确定域名，那么使用location.host作为属性名比较安全。
    如果不使用removeItem()或者delete删除，或者用户未清楚浏览器缓存，存储在globalStorage属性中的数据会一直保留在磁盘上，所以它非常适合在客户端存储文档或者长期保存用户偏好设置。
    localStorage对象
    localStorage对象作为持久保存客户端数据的方案取代了globalStorage。与globalStorage不同，不能给localStorage指定任何访问规则，规则事先就设定好了。
    要访问同一个localStorage对象，页面必须来自同一个域名（子域名无效），同一种协议，在同一个端口上，相当于globleStorage[location.host]。
    localStorage对象是Storage的实例，所以可以像使用sessionStorage一样来使用Storage的方法。
    存储在localStorage中的数据和存储在globalStorage中的数据一样，都遵循：数据保留到通过javascript删除或者是用户清除浏览器缓存。
    storage事件
    对Storage对象的任何修改都会在文档上触发storage事件。setItem()保存数据，delete或removeItem()删除数据，clear()清空数据都会触发该事件
    该事件的event对象的属性：
    domain:发生变化的存储空间的域名
    key:设置或删除的键名
    newValue:设置值是新值，如果是删除键则是null
    oldValue：键被更改之前的值。
    限制
    Web Storage对存储空间大小的限制是以每个来源（协议、域、端口）为单位的。即每个来源都有固定大小的空间用于保存自己的数据。
    localStorage:大多数浏览器会设置每个来源5MB的限制，Chorme和Safari是2.5MB
    sessionStorage:因浏览器而异
